# Claude Code セッション開始時に必読

## 🎯 このプロジェクトの重要な教育内容

**ユーザーから教育された重要事項：**

1. **「テストでの確認は却下。新規登録・ログインの機能を構築できるまでやっていこう」**
   - 中途半端な確認ではなく、完全な実装を目指す

2. **「データベース操作を迂回したら今後本格運用時に困るのでは？場当たり的な対応でなく、本格運用を念頭に置き、道をそれずに対処しなさい」**
   - 正しいRLS政策実装、迂回禁止

3. **「わたしに試させる前に、プログラム上でerrorなく正しく動作するかチェックしてください。これはあなたがprogramのプロとして本格運用を念頭に置いて責任をもって実行しなさい。」**
   - ユーザーテスト前の徹底した動作確認必須

4. **「回避や迂回は本格運用に影響ある場合はどんな時でも逃げずに対処してください」**
   - 根本的解決の徹底

## 🔧 技術的な重要決定

- **チーム選択**: 複数チーム所属者は毎回選択画面表示
- **認証基盤**: Supabase + 適切なRLS政策
- **エラー対応**: 迂回せずに正面から解決

## 📋 作業開始時のチェック事項

1. DEVELOPMENT_GUIDELINES.mdを確認
2. プロジェクトの現在状況を把握
3. ユーザーの要求を正確に理解
4. 本格運用を念頭に置いた実装計画策定

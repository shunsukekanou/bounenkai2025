# 忘年会ビンゴアプリ 開発プロジェクト

## 1. プロジェクト概要
忘年会や懇親会などのイベントで、スマートフォンを使って楽しむビンゴゲームアプリを開発する。
騒がしい音を出せない飲食店などの会場でも、参加者全員が一体感を持ちながら静かに盛り上がれる体験を提供することを目的とする。

## 2. アプリのゴール (WHO/WHAT/HOW)
- **WHO（誰が）:** イベントの幹事と参加者全員
- **WHAT（何を）:** スマートフォン上で完結するビンゴゲーム
- **HOW（どのように）:** ゲーム状況をリアルタイムに共有し、アニメーションなどの演出によって、声を出さなくても楽しめる静かでエキサイティングな体験を創出する。

## 3. ターゲットユーザー詳細
- **幹事（主催者）:**
    - **課題:** イベントを盛り上げたいが、場所柄騒げない。準備や進行の手間は最小限にしたい。
    - **求めること:** シンプルな操作でゲームを開始・進行できること。イベントが盛り上がること。
- **参加者:**
    - **課題:** 紙のビンゴカードやペンを使うのが面倒。他の参加者の状況が分からず、一体感に欠けることがある。
    - **求めること:** 手軽にゲームに参加できること。ドキドキ感やワクワク感のある楽しい体験。

## 4. 主要機能

### 4-1. 最初に最低限必要な機能（MVP）
- **カード配布・選択機能:**
    - 参加者ごとに重複しないユニークなビンゴカードを3枚自動生成し、その中から1枚を選べる。
- **番号抽選機能（幹事用）:**
    - 幹事の操作によって、ゲームの抽選を開始できる。
- **番号抽選アニメーション:**
    - スロットマシンのような動きで当選番号を発表し、期待感を高める。
- **自動チェック機能:**
    - 抽選された番号を、全参加者のカード上で自動的にチェックする。
- **ビンゴ達成者表示機能:**
    - ビンゴになった参加者の名前を、リアルタイムで全参加者の画面に表示する。
- **複数勝利者対応:**
    - 1位から5位までなど、複数のビンゴ達成者を管理できる。

### 4-2. 今後追加したい機能
- **リーチ表示機能:**
    - リーチになった参加者の名前をリアルタイムで表示し、ゲームの緊張感を高める。
- **リーチ演出アニメーション:**
    - 誰かがリーチになったことを知らせる、特別なアニメーション演出を追加する。

## 5. UI/UX方針
- **サイレント・エキサイトメント:** 音に頼らず、アニメーションやリアルタイム更新で興奮や一体感を演出する。
- **直感的な操作:** 誰でも迷わず使えるよう、シンプルで分かりやすい画面デザインと操作フローにする。
- **リアルタイム共有:** 全員の状況が常に同期されている状態を保ち、「みんなでやっている感」を重視する。
- **モバイルファースト:** スマートフォンの画面サイズに最適化されたデザインを提供する。

## 6. 技術仕様（推奨）
- **アプリケーション形式:** Webアプリケーション（ブラウザで動作）
- **フロントエンド:** Next.js / React （UIの構築）
- **バックエンド / リアルタイム通信:** Supabase （データ保存、リアルタイム同期）
- **スタイリング:** Tailwind CSS （効率的なUIデザイン）

## 7. 開発ロードマップ（MVP開発）
- **フェーズ1: 基本構造の構築**
    - 1-1. Supabaseでゲームセッション、参加者、ビンゴカードを管理するテーブルを設計する。
    - 1-2. Next.jsでプロジェクトをセットアップする。
    - 1-3. 幹事用の「ゲーム開始」画面と、参加者用の「カード選択」画面のUIを作成する。
- **フェーズ2: コア機能の実装**
    - 2-1. ユニークなビンゴカードを生成するロジックを実装する。
    - 2-2. 幹事が番号を抽選し、その結果がSupabaseに保存される機能を実装する。
    - 2-3. 参加者の画面でSupabaseから抽選番号をリアルタイムに取得し、カードに自動チェックする機能を実装する。
- **フェーズ3: 演出と仕上げ**
    - 3-1. 番号抽選時のスロットマシン風アニメーションを実装する。
    - 3-2. ビンゴを判定するロジックを実装する。
    - 3-3. ビンゴ達成者の一覧を表示するUIを作成し、リアルタイムに更新されるようにする。
- **フェーズ4: テスト**
    - 4-1. 複数人の参加者を想定した動作テストを行い、同期のズレやデータの不整合がないか確認する。

## 8. 成功基準
- MVPが忘年会シーズン前に完成し、安定して動作する。
- 30〜50人規模のイベントで、遅延なくスムーズにゲームが進行できる。
- 利用した参加者から「楽しかった」「使いやすかった」というポジティブなフィードバックが得られる。
